❓ Help
What is BLE?

Bluetooth Low Energy — wireless protocol for sensors, wearables, IoT. Low power, short range.

Why can't I scan?

Chrome or Edge required. Must be HTTPS or localhost. Accept Bluetooth permission.

What is a characteristic?

A data slot in a BLE service. You can read it, write to it, or subscribe to notifications.

Is data logged?

Yes — Python backend logs all events to /logs/ as CSV and JSON Lines automatically.

How to start backend?

cd backend && pip install -r requirements.txt && python main.py

1

2

3

Click Scan. Optionally filter by name or RSSI.

4

Click Connect on a device, then Discover for its services.

5

Select service + characteristic → Read / Write / Subscribe.

6

Watch the real-time chart. Export logs from Settings.

📡 BLE Protocol

GATT-based. Services group Characteristics with properties: read, write, notify, indicate.

🔑 UUIDs

16-bit standard (e.g. 0x2A37 = Heart Rate) or 128-bit custom. Both supported.

🐍 Backend

FastAPI + bleak. WS: ws://localhost:8000/ws. REST: /api/. Logs: /logs/. Linux · macOS · RPi.

📜 Log

TX (blue) = sent, RX (cyan) = received. Filter by type. Export as .txt.

📡 Lesson 1: What is Bluetooth?

Bluetooth is a way for devices to talk to each other using invisible radio waves. It works like a walkie-talkie but for your gadgets!

BLE (Bluetooth Low Energy) is a special version that uses very little battery — perfect for sensors and wearables.

📱 Phone ──radio waves──▶ ⌚ Watch 📱 Phone ──radio waves──▶ 🎧 Headphones 📱 Phone ──radio waves──▶ 🌡️ Temperature Sensor

BLE uses the 2.4 GHz frequency — the same band as WiFi and microwave ovens!

🔑 Lesson 2: What is a byte?

A byte is 8 bits. Each bit is a 0 or 1. A byte can hold a number from 0 to 255.

1 bit = 0 or 1 1 byte = 8 bits = 0000 0000 to 1111 1111 1 byte = 0 to 255 in decimal 1 byte = 00 to FF in hexadecimal Example: the letter "A" = 0x41 = 65 = 0100 0001

When you read a BLE characteristic, you get bytes. The dashboard shows them as hex (like FF 00 A3) and also tries to decode them as text or numbers.

🏗️ Lesson 3: GATT — How BLE organizes data

BLE devices organize their data using GATT (Generic Attribute Profile). Think of it like a folder structure:

📦 Device ├── 📂 Service: "Heart Rate" (0x180D) │ ├── 📄 Characteristic: "Heart Rate Measurement" (0x2A37) │ └── 📄 Characteristic: "Body Sensor Location" (0x2A38) ├── 📂 Service: "Battery" (0x180F) │ └── 📄 Characteristic: "Battery Level" (0x2A19) └── 📂 Service: "Device Info" (0x180A) ├── 📄 Manufacturer Name └── 📄 Firmware Version

Each characteristic has properties: Read (R), Write (W), Notify (N). Notify means the device pushes updates to you automatically!

📻 Lesson 4: Radio waves and frequency

Everything wireless uses radio waves — invisible waves that travel at the speed of light. Different frequencies are used for different things:

FM Radio = 88–108 MHz (music) WiFi = 2.4 GHz + 5 GHz (internet) BLE = 2.4 GHz (sensors) Cell/4G/5G = 700–2600 MHz (phone calls) MHz = millions of waves per second GHz = billions of waves per second

BLE uses 40 channels in the 2.4 GHz band. 3 channels (37, 38, 39) are for advertising — that's how devices announce "I'm here!". The other 37 channels are for data, and BLE hops between them to avoid interference.

📏 Lesson 5: RSSI — measuring invisible signals

RSSI (Received Signal Strength Indicator) tells you how strong a radio signal is. It's measured in dBm (decibels relative to milliwatt).

The values are always negative — closer to 0 = stronger signal:

-30 dBm = EXCELLENT (almost touching) -50 dBm = GOOD (same room) -70 dBm = OK (next room) -90 dBm = WEAK (far away) -100 dBm = LOST (out of range)

The radar view uses RSSI to estimate distance. Try walking around with a BLE device and watch the signal change!
