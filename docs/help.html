Help & FAQ

Answers to common questions and troubleshooting

General

What is BLE?

Bluetooth Low Energy (BLE) is a wireless protocol designed for low-power devices like fitness trackers, smart watches, temperature sensors, micro:bit, and IoT gadgets. It uses the 2.4 GHz radio band and has a typical range of 10–30 meters.

What is GATT?

Generic Attribute Profile — the way BLE organizes data. A device has Services (groups of related data) and each service has Characteristics (individual data values). Each characteristic has properties like Read, Write, and Notify.

What is a UUID?

A unique identifier for services and characteristics. Standard UUIDs are 16-bit (e.g., 0x2A37 = Heart Rate Measurement). Custom devices use 128-bit UUIDs (e.g., e95d0753-251d-470a-a062-fa1922dfa9a8 for micro:bit accelerometer).

What is RSSI?

Received Signal Strength Indicator — measures signal power in dBm (decibels relative to milliwatt). Values are always negative: closer to 0 = stronger signal. Typical ranges:

RSSISignalDistance
-30 to -50Excellent< 1 meter
-50 to -70Good1–5 meters
-70 to -85Fair5–10 meters
-85 to -100Weak> 10 meters
What is SDR?

Software Defined Radio — a radio receiver where processing is done in software instead of hardware. The HackRF One is a popular SDR device that can receive 1 MHz to 6 GHz. This dashboard uses it to visualize the 2.4 GHz band where BLE and WiFi operate.

What are BLE channels?

BLE uses 40 channels in the 2.4 GHz band:

  • 3 advertising channels (37, 38, 39) — used for device discovery and broadcasting
  • 37 data channels (0–36) — used for connected communication with frequency hopping

The advertising channels are placed between WiFi channels to reduce interference.

Setup & Installation

Which browsers work?

Chrome and Edge support Web Bluetooth. Safari and Firefox do not. The page must be served from http://localhost or HTTPS.

How do I install on Linux / Raspberry Pi? 
sudo apt update
sudo apt install bluetooth bluez python3 python3-pip python3-venv
cd ble-dashboard
bash launch.sh     # choose Install (2), then Start (3)

Add yourself to the bluetooth group for non-root access:

sudo usermod -aG bluetooth $USER
newgrp bluetooth
How do I install on macOS? 
brew install python3    # if not already installed
cd ble-dashboard
bash launch.sh          # choose Install (2), then Start (3)

Accept the Bluetooth permission popup when Chrome asks for it.

What Python version do I need?

Python 3.10 or newer. Check with: python3 --version

Troubleshooting

Scan button doesn't work / no devices found

Check these in order:

  1. Browser: Use Chrome or Edge (not Safari/Firefox)
  2. URL: Must be http://localhost:8000 (not 127.0.0.1)
  3. Bluetooth: Make sure your adapter is powered on
  4. Permission: Click "Allow" when the browser asks for Bluetooth access
  5. Distance: BLE devices must be within ~10 meters
Use the 🎲 Demo button to verify the dashboard works without real devices.
Server won't start / port already in use

The default port is 8000. If it's already in use:

# Find what's using port 8000
lsof -i :8000

# Start on a different port
BLE_PORT=8001 python backend/main.py

# Or use the launcher
bash launch.sh   # go to Configuration (9) → Change port
Connection drops / GATT disconnect

BLE connections can be fragile. Common causes:

  • Device moved out of range
  • Device battery is low
  • Another app is connected to the same device
  • Bluetooth adapter is busy

Enable Auto-reconnect in the Scanner to automatically reconnect on disconnect.

Can't write to a characteristic

Check that the characteristic has the W (Write) property badge. Some characteristics are read-only. Also check:

  • The device is still connected (green status dot)
  • Your value format is correct (hex pairs like FF00 or plain text)
  • Some characteristics require specific byte lengths
Bluetooth is rfkill-blocked (Linux) 
sudo rfkill unblock bluetooth
sudo systemctl restart bluetooth

Verify with: rfkill list bluetooth — should show "Soft blocked: no"

HackRF not detected
  1. Check USB connection: lsusb | grep -i hackrf
  2. Install tools: sudo apt install hackrf (Linux) or brew install hackrf (macOS)
  3. Test: hackrf_info — should show firmware version
  4. Linux permissions: sudo chmod 666 /dev/bus/usb/... or add udev rules
You can always use Demo Mode without any hardware.
Dashboard looks broken / old version cached

Click the 🔄 button in the dashboard header to clear cache and reload. Or:

Ctrl+Shift+R    (hard reload in Chrome)
Cmd+Shift+R     (macOS)
