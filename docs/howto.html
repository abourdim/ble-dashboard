1. Scan for BLE Devices

  1. Open the dashboard at http://localhost:8000
  2. The BLE Scanner section is open by default at the top
  3. Click the Scan button
  4. Your browser asks for Bluetooth permission — click Allow
  5. Devices appear in the list with name, signal strength bars, and RSSI value

Filter devices

  • Name filter: Type part of a device name (e.g., "HC-05" or "sensor")
  • Min RSSI: Set to -60 to only show strong (close) signals
  • Hide unknown: Check this to hide devices without a name
No devices nearby? Click the 🎲 button for demo mode with 6 simulated devices.

2. Connect and Explore Services

  1. After scanning, click Connect on any device
  2. The Characteristics panel opens automatically
  3. Click Discover to load all services and characteristics
  4. Services appear as expandable groups with color-coded headers
  5. Each characteristic shows its UUID and property badges: R (read), W (write), N (notify)
Known characteristics are automatically decoded: Battery Level shows as "85%", Heart Rate as "72 BPM", Temperature as "23.5°C", etc.

3. Read and Write Characteristics

Reading

  1. Find a characteristic with the R badge
  2. Click the R button next to it
  3. The value appears in hex, UTF-8 text, and integer formats

Writing

  1. Find a characteristic with the W badge
  2. Click the W button
  3. Type your value — the dashboard auto-detects hex (e.g., FF00) vs text
  4. Click Send
Use Read All to read every readable characteristic at once.

4. Subscribe to Notifications

  1. Find a characteristic with the N badge
  2. Click the N button to subscribe
  3. Values stream in real-time and appear in the chart below
  4. Click N again to unsubscribe

Watch Mode

Click Watch to automatically subscribe to ALL notifiable characteristics and poll all readable ones. Great for getting a full picture of a device.

5. Use the Real-time Chart

  1. Subscribe to a characteristic that sends numeric data
  2. The chart auto-creates a line for each data series
  3. If data arrives as KEY:VALUE format (e.g., temp:23.5), each key gets its own line

Controls

  • Window: How many data points to show (default 60)
  • Pause: Freeze the chart while data keeps coming
  • Grid: Toggle background grid lines
  • Export: Download chart data as CSV

6. Use Radar View

  1. Start a scan
  2. Click the 📡 radar toggle button
  3. Devices appear as blips on concentric rings based on estimated distance
  4. Color indicates proximity: green (<1m), yellow (~5m), orange (~10m), red (>10m)
  5. Click a blip to see device details

7. Packet Analyzer

Opening the analyzer

The packet analyzer is built into the service explorer. Every BLE operation (read, write, notify) is captured automatically.

Filtering packets

  • Type in the filter box to search by text
  • Use field queries: type:notify, uuid:e95d, len>5

Hex dump

Click any packet to see its detail tree (Frame → BLE → GATT → Value) and hex dump with ASCII sidebar.

Double-click a packet to mark it. Use "Follow" to filter to just one characteristic's packets.

8. Record and Play Macros

  1. In the nRF Tools section, find the Macro Recorder
  2. Click Record
  3. Perform BLE operations (read, write, subscribe)
  4. Click Stop
  5. Click Play to replay the same sequence

You can Save macros for later and Export them as JSON files.

9. SDR Spectrum Analyzer

  1. Open the SDR Spectrum (2.4 GHz) section
  2. Make sure Demo Mode is checked (unless you have a HackRF One)
  3. Click Start

What you see

  • Waterfall: Time flows downward. Colors show signal strength (blue = weak, yellow = strong)
  • Channel Bars: 40 BLE channels. Red bars = advertising channels (37, 38, 39)
  • BLE vs WiFi: Comparison chart showing how WiFi channels overlap with BLE
Check the WiFi Overlay toggle to see WiFi channel regions highlighted on the charts.

10. Multi-device Connections

  1. Connect to a first device normally
  2. Scan again and connect to another device
  3. Open the Connected Devices section to see all active connections
  4. Up to 5 simultaneous BLE connections are supported

11. Debug and Export

Connection Timeline

Visual history of all events (connect, disconnect, read, write, error) with timestamps and color-coded dots.

Read Latency Report

See how long each read operation takes, with bar visualization. Useful for identifying slow characteristics.

Debug Snapshot

One-click JSON export of the full dashboard state: timeline, data rates, latency, services, errors. Share with others for troubleshooting.

Log Export

From Settings, click Export logs from backend to download CSV and JSONL log files.

12. Set Up on Raspberry Pi

  1. Install system packages: 
    sudo apt update
sudo apt install bluetooth bluez python3 python3-pip python3-venv
  2. Add your user to the bluetooth group: 
    sudo usermod -aG bluetooth $USER
newgrp bluetooth
  3. Clone the project and run the launcher: 
    cd ble-dashboard
bash launch.sh
  4. Choose Install (2), then Start (3)
  5. Open http://raspberry-pi-ip:8000 from another device on the same network
If Bluetooth is blocked: sudo rfkill unblock bluetooth

13. Use Real HackRF One Hardware

  1. Install HackRF tools: 
    # Linux / Raspberry Pi
sudo apt install hackrf

# macOS
brew install hackrf
  2. Plug in your HackRF One USB device
  3. Verify it's detected: hackrf_info
  4. In the SDR section, uncheck Demo Mode
  5. Click Start — the dashboard runs hackrf_sweep to capture real spectrum data
The dashboard sweeps 2400–2500 MHz (the BLE/WiFi band) and maps all 40 BLE channels to their exact frequencies.
